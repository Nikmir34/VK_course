iOS Guidelines

Guidelines – набор рекомендаций от создателей платформы, благодаря которым
приложения сторонних разработчиков выглядят единообразно.

1 Приложение не может использовать "непубличные API". Это означает, что
вы не можете использовать некоторые функции, которые платформа использует
для своих собственных приложений.
2 Приложение не может перепрограммировать контрольные клавиши
устройства, не предназначенные для подобного использования (к примеру,
использовать кнопку громкости для фотосъемки).
3 Приложение не должно иметь доступ к файлам, находящимся вне
приложения, без разрешения пользователя.
4 Приложение не должно иметь доступа к к папкам и файлам вне директорий
контейнера и документов.
5 Приложение не может загружать код для установки без согласия
пользователя.
6 Приложение может обновляться только через магазин приложений.
7 После загрузки приложение должно быть рабочим. Оно не должно
отключиться через несколько дней.
8 Приложение не может быть "опытной", "бета", "демо" или "тест"-версией.
Названия продуктов Apple должны быть указаны без опечаток (iPhonez –
неверно).
10 Если приложение использует сеть, оно не пользуется ей через сторонние
(не-Apple) браузеры.
11 В приложении не должны упоминаться сторонние платформы (к примеру,
"Также доступно для Android!").
12 Приложение не может использовать устаревшие интерфейсы (например,
колесо iPod).
13 Многозадачная функциональность приложения может быть доступна
только для его исходных целей (то есть для VoIP, проигрывание аудио,
определение местоположения, завершение задач, локальные уведомления, и т.
п.). Это значит, что в общем случае приложение не может работать в фоновом
режиме и должно быть закрыто, если оно не используется.
14 В приложении должна быть хоть какая-то функциональность. Оно не может
состоять из одной странички и текста, не может быть просто песней, фильмом
или книгой – для этого есть другие платформы.
15 Функциональность должна соответствовать описанию в магазине
приложений.
16 В целом приложение должно быть "достойным". Откровенные материалы –
секс, насилие, наркотики, алкоголь, табак – не должны в нем
демонстрироваться, и оно не должно отзываться об отдельных этнических или
религиозных группах уничижительно.
17 Приложение должно быть честным. Его описание должно быть правдивым,
и вся функциональность должна работать, как описано. Если приложение дает
диагностическую информацию, она должна быть надежной. Это также касается
жанра и категории в магазине. Иконки приложения должны соответствовать и
подходить ему.
18 Приложение не может ограничивать пользователей в выборе геолокации
или мобильного оператора.
19 Приложение не может рассылать спам, распространять вирусы, или
использовать другие платформы Apple (например, Game Center/Push
Notifications) с этими целями.
20 Приложение должно стремиться хранить минимальное количество данных
в iCloud. Все, хранящееся в iCloud, должно быть создано пользователем.
Информация, которую можно загрузить или восстановить, не должна попадать в
iCloud.
21 Приложение не может использовать геолокационные службы без
разрешения.
22 Все ссылки в коде приложения должны работать.
23 Приложение не может использовать местонахождение пользователя без
разрешения.
24 Геолокационные службы не могут использоваться для автономного
контроля транспортных средств или вызова служб скорой помощи.
25 Приложение не может использовать уведомления без согласия
пользователя.
26 Уведомления должны рассылаться через Apple Push Notification API, при
помощи APN ID.
27 Уведомления не должны содержать личных данных.
28 Приложение не может распространять личную информацию пользователей
(например, ID игрока) через Game Center.
29 Рекламные баннеры должны скрываться, если реклама недоступна.
30 Приложение должно уважать авторские права Apple и других сторон.
31 Механизм встроенных покупок не может использоваться для приобретения
товаров и услуг, использующихся вне приложения.
32 Механизм встроенных покупок не может использоваться для сбора средств
на благотворительность (для этого есть SMS).
33 Механизм встроенных покупок не может использоваться для покупки
лотерейных билетов напрямую из приложения.
34 Приложения, которые поощряют пользователя использовать устройство
так, что оно может быть повреждено, не будут одобрены.
35 Приложение не может запрашивать личные данные (например, email)
пользователя, чтобы функционировать.
